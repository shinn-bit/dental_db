# 引き継ぎ.md

Claude Codeへ移行するための引き継ぎ情報。

## 現在の状態

- アプリはNext.jsベースの院内ナレッジ/資料庫アプリ。
- 資料庫にPDF、Word、テキスト、画像、動画をアップロードできる。
- OCR/要約/RAG同期はAWS側で処理する。
- AIチャットはBedrock Knowledge Baseを参照して回答する。
- 旧「本棚」「種類」「診療領域」「読む人」は廃止済み。
- 旧 `/manuals` は `/repository` へリダイレクトする。

## 重要な最近の変更

### 資料庫への整理

- `components/manuals-manager.tsx` は `components/file-repository-manager.tsx` に移行済み。
- `lib/manuals.ts` は削除され、`lib/file-assets.ts` が現在のメタデータ定義。
- `StoredFileMetadata` と `normalizeFileMetadata()` が中心。
- 旧分類フィールドは保存・表示しない。

### 要約ルール

現在の要約ルールは以下。

- 9項目要約は廃止。
- 目次、章、節、見出しごとに要約。
- 各見出しは400字程度。
- 長文資料はチャンクごとに要約し、最後はLLMに再統合させない。
- 各チャンクの章別要約を単純連結する。

この仕様にした理由:
- 最終統合LLMに渡すと、固定9項目要約へ戻ることがあったため。

### マニュアル作成機能は未実装

次に実装予定の機能。

- AIチャット画面を左右分割する。
- 左: AIチャット。
- 右: マニュアル生成。
- 最初の出力形式はWord `.docx`。
- その後PowerPoint `.pptx`。
- マニュアル生成時は9項目構成を使う予定。
- ただし資料単体の要約は9項目にしない。

## 処理フロー

### 新規アップロード

1. `/api/upload-url` で署名付きS3 PUT URLを作成。
2. ブラウザがS3へ実体ファイルをPUT。
3. `/api/files` にメタデータ保存。
4. PDF/画像/テキスト系はPrepare workflow開始。
5. LambdaがOCR/Textractを実行。
6. 抽出テキストをS3へ保存。
7. KB用Markdownを `kb/{id}.md` へ保存。
8. Bedrock KB同期ジョブを開始。

### 要約作成

1. `/api/files/[id]/summary` POST。
2. Summary workflow開始。
3. Lambdaが `extractedTextKey` の本文を読む。
4. 章・節・見出しごとに400字程度で要約。
5. `summaries/{id}.md` へ保存。
6. `kb/{id}.md` にも要約Markdownを保存。
7. メタデータに `summaryStatus: completed`、`summaryMode: section` を保存。

### AIチャット

1. `/api/files` から資料候補を読み込む。
2. 選択資料がある場合、`knowledgeBaseKey` / `summaryKey` / `extractedTextKey` からSource URIフィルタを作る。
3. Bedrock Knowledge BaseのRetrieveAndGenerateで回答。

## 移行時に必要な確認

Claude Code移行直後に以下を確認する。

```powershell
node --version
npm --version
aws sts get-caller-identity --profile dental-dev
npm run lint
npm run build
```

Lambdaを触る場合:

```powershell
python -m py_compile lambda\summary_worker\handler.py
```

AWS反映:

```powershell
Compress-Archive -LiteralPath .\lambda\summary_worker\handler.py -DestinationPath .\lambda\summary_worker.zip -Force
aws lambda update-function-code --function-name dental-summary-worker-dev --zip-file fileb://lambda/summary_worker.zip --profile dental-dev --region ap-northeast-1
aws lambda get-function-configuration --function-name dental-summary-worker-dev --profile dental-dev --region ap-northeast-1 --query "{LastUpdateStatus:LastUpdateStatus,LastModified:LastModified,CodeSha256:CodeSha256}"
Remove-Item -LiteralPath .\lambda\summary_worker.zip -Force
```

## 注意すべき落とし穴

- Git pushだけではLambdaは更新されない。
- Amplifyに反映されるのはNext.js側。Lambdaは別更新。
- SSO期限切れ時はAWS CLIが失敗する。
- `S3_MANUAL_PREFIX=manuals/` は古い名前だが、実体キー互換のため維持する。
- 既存資料の古い9項目要約は自動では再作成されない。
- 新しく作成される要約から章別要約ルールが適用される。
- 図・画像中心のPDFは通常OCRだけでは内容把握に限界がある。Vision処理は今後対応予定。
